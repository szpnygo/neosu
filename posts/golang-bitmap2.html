Golang

上一节中，我们实现了一个基本的BitSet，现在有个需求，计算BitSet中1的个数。

这个需求使用场景很多，比如统计在线用户数量等。之前的实现完成了记录，本节会完成统计。

这个需求其实就是计算汉明重量

汉明重量

汉明重量是一串符号中非零符号的个数。因此它等同于同样长度的全零符号串的汉明距离。在最为常见的数据位符号串中，它是1的个数。

汉明重量是以理查德·卫斯里·汉明的名字命名的，它在包括信息论、编码理论、密码学等多个领域都有应用。

（一）遍历法

比较简单的方法就是直接遍历法，就是遍历位数组中的每个二进制位，并在遇到值为1的二进制位时，将计数器的值增一

遍历算法虽然实现起来简单，但效率非常低

例如，假设要检查的位数组的长度为100MB，那么按1MB=1000000Byte=8000000bit来计算，使用遍历算法检查长度为100MB的位数组将需要执行检查操作八亿次（100*8000000）

（二）查表法

对于一个有序集合来说，集合元素的排列方式是有限的。

所以我们可以创建一个表，表的键为某种排列的位数组，而值是其中1的数量

比如我们一次读取8位，然后根据这8个位的值进行查表，直接查对应包含了多少个1

这样我们可以一次检查8位，对比遍历法，等于快了8倍

100MB的数组，只需要1亿次查询。

但是这个方法不是完全的，它的提升是有限的。提升位数，可以减少计算次数。

位数越大，这个表所需要的内容空间也就越大。32位的表可能需数G的内存空间消耗。

查表法就是典型的，用空间换时间的方法。

（三）variable-precision SWAR算法

因为汉明重量经常被用于信息论、编码理论和密码学，所以研究人员针对计算汉明重量开发了多种不同的算法，一些处理器甚至直接带有计算汉明重量的指令，而对于不具备这种特殊指令的普通处理器来说

目前已知效率最好的通用算法为variable-precision SWAR算法

该算法通过一系列位移和位运算操作，可以在常数时间内计算多个字节的汉明重量，并且不需要使用任何额外的内存。

比如我使用的，一次计算64位，只需要4次操作。比8位查表法快了8倍。

variable-precision SWAR

不过我对SWAR算法的原理还没有太多研究，只是先根据网上的算法，实现一个uint64的SWAR算法

func (set *BitSet) Count() uint64 {
	var count uint64
	for _, b := range set.data {
		count += Swar(b)
	}
	return count
}

func Swar(i uint64) uint64 {
	i = (i & 0x5555555555555555) + ((i >> 1) & 0x5555555555555555) // 步骤1
	i = (i & 0x3333333333333333) + ((i >> 2) & 0x3333333333333333) // 步骤2
	i = (i & 0x0F0F0F0F0F0F0F0F) + ((i >> 4) & 0x0F0F0F0F0F0F0F0F) // 步骤3
	i = (i * 0x0101010101010101) >> 56                             // 步骤4
	return i
}

	bitset.Swar(uint64(1357913)) //10

}
