Redis Bitmap位图介绍与操作

2021-05-08
7分钟阅读时长

位图介绍

Redis的位图(bitmap)是由多个二进制位组成的数组，数组中的每个二进制位都有与之对应的偏移量，用户通过这些偏移量可以对位图中指定一个或多个二进制位进行操作。

下图展示了一个包含8个二进制位的位图，这个位图储存的值为01001010

索引01234567
位01001010

Redis为位图提供了一系列的操作命令:

  • setbit 为位图上指定偏移量上的二进制位设置值
  • getbit 获取位图指定偏移量上的二进制值
  • bitcount 统计位图中有多少个值被设为1
  • bitop 对一个或多个位图进行逻辑操作，如逻辑并、或、异以及逻辑非运算
  • bitpos 查找位图中第一个被设置为指定值的二进制位并返回其偏移量

优势和限制

优势

  • 基于最小单位bit(一个字节8位，位是最小单位)进行储存，非常节省空间
  • 设置的时候，时间复杂度是$O(1)$、读取的时候时间复杂度是$O(n)$，所以操作速度很快
  • 二进制数据的储存，进行运算的审核非常快
  • 方便扩容

限制

  • bitmap有长度限制，被限制在$2^{32}$位，等于$512mb$
  • key的位数越长，读取速度越慢，因为读取的时候时间复杂度是$O(n)$

偏移量是${2^{32}-1}$的时候，需要分配内存$512mb$
$2^{32}$是42亿，也就是位图可以轻易存下上亿的内容，不过太长会导致读取太慢，一般太长也会进行分片操作
根据官方文档提供的数据，设置不同长度位图所需要的时间

  • $2^{32}-1$大约需要$300ms$
  • $2^{30}-1$大约需要$80ms$
  • $2^{28}-1$大约需要$30ms$
  • $2^{26}-1$大约需要$8ms$
    一旦完成第一个分配，对同一key的后续调用将没有分配开销

bitmap结构

Redis使用字符串对象来表示位数组，因为字符串你对象使用的SDS数据结构是二进制安全的，所以可以直接使用SDS结构来保存维数组，并使用SDS结构的操作函数来处理位数组。

struct __attribute__ ((__packed__)) sdshdr64 {
    uint64_t len; /* used */
    uint64_t alloc; /* excluding the header and null terminator */
    unsigned char flags; /* 3 lsb of type, 5 unused bits */
    char buf[];
};

bitmap操作

setbit

通过使用$setbit$命令，用户可以为位图指定偏移量上的二进制设置值:

setbit key offset value

$setbit$命令在对二进制设置之后，将返回二进制位被设置之前的旧值作为结果。

offset最大值是$2^{32}-1$，且offset只能为正数

当用户执行$setbit$命令尝试对一个位图进行设置的时候，如果位图不存在，或者位图当前的大小无法满足用户想要执行的设置操作，那么Redis将对被设置的位图进行扩展，使得位图可以满足用户的设置请求。

因为Redis对位图的扩展操作是以字节为单位进行的，所以扩展之后的位图包含的二进制位数量可能会比用户要求的稍微多一些，并且在扩展位图的同时，Redis还会将所有未被设置的二进制位的值初始化为0。

127.0.0.1:6379> setbit bitmap01 0 1
(integer) 0         #二进制位原来的值位0

127.0.0.1:6379> setbit bitmap01 0 1
(integer) 1         #二进制位原来的值位0

127.0.0.1:6379> setbit bitmap01 2 1
(integer) 0         #bitmap01此时的值  10100000  左侧第一个为索引0
#初始化1个字节，所以有8位，其他的都是0

127.0.0.1:6379> setbit bitmap01 1000000000000 1
(error) ERR bit offset is not an integer or out of range 
#offset最大值是2的32次方-1，且offset只能为正数

127.0.0.1:6379> strlen bitmap01
(integer) 1         
#bitmap01此时的长度是1，虽然我们已经设置了offset 2 为1
#但是这里的offset是按bit位偏移的，strlen返回的是字节长度，1个字节等于8位

127.0.0.1:6379> setbit bitmap01 8 1
(integer) 0         #设置偏移8的二进制为1

127.0.0.1:6379> strlen bitmap01
(integer) 2         #长度为2个字节

getbit

通过使用$getbit$命令，用户可以获取位图指定偏移量上的二进制值:

getbit key offset

和$setbit$一样，offset最大值是$2^{32}-1$，且offset只能为正数

#bitmap 上节中已经设置为1010000010000000
127.0.0.1:6379> getbit bitmap01 0
(integer) 1
127.0.0.1:6379> getbit bitmap01 1
(integer) 0
127.0.0.1:6379> getbit bitmap01 2
(integer) 1
127.0.0.1:6379> getbit bitmap01 8
(integer) 1
127.0.0.1:6379> getbit bitmap01 9
(integer) 0
127.0.0.1:6379> getbit bitmap01 100
(integer) 0         #超出当前位图最大偏移量的，返回0作为结果

bitcount

通过$bitcount$命令统计为位图中值为1的二进制位数量：

bitcount key [start end]

对于值为1010000010000000的位图bitmap01。可以这样统计它有多少个二进制位被设置成了1

127.0.0.1:6379> bitcount bitmap01
(integer) 3

start和end可以对指定字节范围的二进制进行统计：

#当前bitmap01位图10100000 10000000 
#增加一个字节
127.0.0.1:6379> setbit bitmap01 16 1
(integer) 0
127.0.0.1:6379> strlen bitmap01
(integer) 3

#设置后为3个字节，10100000 10000000 10000000

127.0.0.1:6379> bitcount bitmap01
(integer) 4
127.0.0.1:6379> bitcount bitmap01 0 0
(integer) 2
127.0.0.1:6379> bitcount bitmap01 0 1
(integer) 3
127.0.0.1:6379> bitcount bitmap01 0 2
(integer) 4
127.0.0.1:6379> bitcount bitmap01 2 2
(integer) 1  
#可以看到，需要特别注意的是，start和end的偏移量是字节（8位）

再次提醒，$bitcount$命令的start参数和end参数定义的是字节偏移量范围，而不是二进制位偏移量范围

$bitcount$命令的start参数和end参数的值除了可以是正数之外，还可以是负数。

127.0.0.1:6379> setbit bitmap01 4 1
(integer) 0
127.0.0.1:6379> setbit bitmap01 18 1
(integer) 0
#当前bitmap01位图10101000 10000000 10100000

127.0.0.1:6379> bitcount bitmap01
(integer) 6  #当前二进制位为1的数量是6

127.0.0.1:6379> bitcount bitmap01 -1 -1
(integer) 2  #统计倒数第一个字节
127.0.0.1:6379> bitcount bitmap01 -2 -2
(integer) 1  #统计倒数第二个字节
127.0.0.1:6379> bitcount bitmap01 0 -2
(integer) 4  #偏移字节0到倒数第2个

bitpos

通过$bitpos$命令在位图中查到第一个被设置为指定值的二进制位，并返回这个二进制位的偏移量：

bitpos key bit [start] [end]

#当前bitmap01位图10101000 10000000 10100000
127.0.0.1:6379> bitpos bitmap01 1   #查到第一个被设置为1的二进制位偏移量
(integer) 0  

127.0.0.1:6379> bitpos bitmap01 0   #查到第一个被设置为0的二进制位偏移量
(integer) 1

127.0.0.1:6379> bitpos bitmap01 1 1 1  
(integer) 8     #在第二个字节中查到第一个被设置为1的二进制位偏移量
127.0.0.1:6379> bitpos bitmap01 0 1 1  
(integer) 9     #在第二个字节中查到第一个被设置为0的二进制位偏移量

127.0.0.1:6379> bitpos bitmap01 1 -1 -1
(integer) 16    #在倒数第一个字节中查到第一个被设置为0的二进制位偏移量

$bitpos$命令返回的偏移量为12，这个偏移量是被找到的二进制位在整个位图中所处的偏移量，而不是它在第二个字节中所处的偏移量。换句话说，即使我们指定了$bitpos$命令要查找的字节范围，$bitpos$命令在返回结果时也只会返回目标二进制位在整个位图中所处的绝对偏移量，而不是这个二进制位在指定字节范围内的相对偏移量。

127.0.0.1:6379> bitpos bitmap02 1  #key不存在返回-1
(integer) -1
127.0.0.1:6379> bitpos bitmap01 1 100  #偏移量超出边界
(integer) -1


127.0.0.1:6379> setbit bitmap03 7 0  #设置一个全为0的位图00000000
(integer) 0
127.0.0.1:6379> bitpos bitmap03 1    #在这个位图中查找1，返回-1
(integer) -1


#设置bitmap04的所有位都是1 11111111
127.0.0.1:6379> setbit bitmap04 0 1
(integer) 0
127.0.0.1:6379> setbit bitmap04 1 1
(integer) 0
127.0.0.1:6379> setbit bitmap04 2 1
(integer) 0
127.0.0.1:6379> setbit bitmap04 3 1
(integer) 0
127.0.0.1:6379> setbit bitmap04 4 1
(integer) 0
127.0.0.1:6379> setbit bitmap04 5 1
(integer) 0
127.0.0.1:6379> setbit bitmap04 6 1
(integer) 0
127.0.0.1:6379> setbit bitmap04 7 1
(integer) 0

127.0.0.1:6379> bitpos bitmap04 0
(integer) 8     
#虽然超出了位图范围，但是Redis会把位图中不存在的二进制位值看做0
#所以bitpos在超出范围后，会继续查找，并返回偏移量8

bitop

用户可以通过$bitop$命令，对一个或多个位图执行指定的二进制位运算，并将运算结果存储到指定的键中：

bitop operation result_key key [bitmap ...]

operation参数的值可以是AND、OR、XOR、NOT中的任意一个，这4个值分别对应逻辑并、逻辑或、逻辑异或和逻辑非4种运算，其中AND、OR、XOR这3种运算允许用户使用任意数量的位图作为输入，而NOT运算只允许使用一个位图作为输入。$bitop$命令在将计算结果存储到指定键中之后，会返回被存储位图的字节长度。

127.0.0.1:6379> setbit bitmap05 1 1
(integer) 0
127.0.0.1:6379> setbit bitmap05 3 1
(integer) 0
127.0.0.1:6379> setbit bitmap06 2 1
(integer) 0
127.0.0.1:6379> setbit bitmap06 3 1
(integer) 0

#创建两个位图 
#bitmap05 01010000
#bitmap06 00110000

# 执行and操作，并把结果保存在result1中，and操作是都为1则为1，都为0则为0
127.0.0.1:6379> bitop and result1 bitmap05 bitmap06
(integer) 1   #and后result1的值是00010000

# 执行or操作，并把结果保存在result2中，or是只要有一个为1则为1，否则为0
127.0.0.1:6379> bitop or result2 bitmap05 bitmap06
(integer) 1   #or后result2的值是01110000

# 执行xor操作，并把结果保存在result3中，xor是两个值不相同，则为1，如果相同则为0
127.0.0.1:6379> bitop xor result3 bitmap05 bitmap06
(integer) 1   #xor后result3的值是01100000

# 执行not操作，并把结果保存在result4中，xor是两个值不相同，则为1，如果相同则为0
# not后只能跟一个位图
127.0.0.1:6379> bitop not result4 bitmap05
(integer) 1   #not后result4的值是10101111


127.0.0.1:6379> setbit bitmap07 2 1
(integer) 0
127.0.0.1:6379> setbit bitmap07 3 1
(integer) 0
127.0.0.1:6379> setbit bitmap07 9 1
(integer) 0
# 创建一个长的位图 00110000 01000000

# 执行or操作，并把结果保存在result5中
# bitmap05 01010000
# bitmap07 00110000 01000000
# 会将长度较短的那个位图中不存在的二进制位的值看作0
127.0.0.1:6379> bitop or result5 bitmap05 bitmap07
(integer) 2  #011100000 01000000 2代表返回的值是2个字节
